Ancestry Source Template

Download link for rmst file = Ancestry (data collections, single and double subject)

For Ancestry, the dilemma to be solved was that the Source Citation provided by Ancestry for most collections were not good enough to use. Of course, one could simply reference the Ancestry source web page, but that wouldn't provide any information on reliability of the source and relies on an Ancestry subscription for access.

The goal for the Ancestry template was to capture all of the info that Ancestry provided and later figure out what the footnote sentences should be. I think that I've now done that as well.

I decided to create two template for Ancestry collections- one for collection that deal with a single person such as birth, death etc and another that deals with two people such as marriage, divorce, etc.

I'll describe the Single person template first.

This is the relevant part of the rmst file. I removed most end tags and empty tags. It is certainly no longer in correct XML syntax.

As an example, use the Ancestry source collection: New York, New York, U.S., Birth Index, 1910-1965. The main page is at:
New York, New York, U.S., Birth Index, 1910-1965

Master Source fields

<Field>

<Type>Text
<Name>Title
<Display>Ancestry database name -short
<Detail>false    This is a Source field. It describes the entire collection.
New York, New York, U.S., Birth Index, 1910-1965

use the Ancestry supplied name, but shortened to remove extraneous info.
Looks at Source Info section of the collection page whose URL is above.
The first sentence is the Ancestry full name-
Ancestry.com. New York, New York, U.S., Birth Index, 1910-1965 [database on-line]. Lehi, UT, USA: Ancestry.com Operations, Inc., 2017.
This field uses the shortened version:
New York, New York, U.S., Birth Index, 1910-1965

<Field>

<Type>Date
<Name>Date
<Display>Last DB info update
<Detail>false    This is a Source field. It describes the entire collection.
2 Jan 2024

This field contains the date on which this source record was entered, or last updated in RM. Use today's date.

<Field>

<Type>Text
<Name>SrcInfo
<Display>Ancestry Source Info
<Detail>false    This is a Source field. It describes the entire collection.
Ancestry.com. New York, New York, U.S., Birth Index, 1910-1965 [database on-line]. Lehi, UT, USA: Ancestry.com Operations, Inc., 2017.

I use the Ancestry supplied name in full. Looks at Source Info section of the collection page whose URL is above. The first sentence is the Ancestry full name-
Ancestry.com. New York, New York, U.S., Birth Index, 1910-1965 [database on-line]. Lehi, UT, USA: Ancestry.com Operations, Inc., 2017.

<Field>

<Type>Text
<Name>OrigData
<Display>Original Data
<Hint>not applicable to all Ancestry collections
<Detail>false    This is a Source field. It describes the entire collection.
New York City Department of Health, courtesy of www.vitalsearch-worldwide.com. Digital Images.

I use the Ancestry supplied info in the Source Info section of the collection page whose URL is above. The second sentence is the Ancestry original data info-
New York City Department of Health, courtesy of www.vitalsearch-worldwide.com. Digital Images.
Note, not all Ancestry collections have this information listed.

<Field>

<Type>Text
<Name>dbID
<Display>Ancestry Database ID
<Hint>Found in URL
<Detail>false    This is a Source field. It describes the entire collection.
61457

The is the Ancestry database number. It can be found in the URL that points to the collection main page.

Citation fields

For citation fields, use the example found at:
example citation

<Field>

<Type>Name
<Name>Name
<Display>Person name
<Detail>true    This is a Citation field.
Walter Otter

This template is for Ancestry collections that have a single person as their focus. This is where that name goes. Given and then Surname.

<Field>

<Type>Date
<Name>BirthDate
<Display>Person's birth date
<Hint>Not necessarily from record. Only for identification.
<Detail>true    This is a Citation field.
29 Dec 1935

Birth date of the focus person.

<Field>

<Type>Date
<Name>EventDate
<Display>Event date
<Detail>true    This is a Citation field.
29 Dec 1935

Date of event from the source. May be same as birth date for birth record.

<Field>

<Type>Text
<Name>CD
<Display>Citation Detail
<Detail>true    This is a Citation field.
<blank>

Some other information about this citation. May be used if there are multiple citations to the same record in a source, or to describe the citation in some way. Usually left blank.

<Field>

<Type>Date
<Name>DateCitation
<Display>Date citation updated
<Detail>true    This is a Citation field.
2 Jan 2024

This field contains the date on which this citation record was entered, or last updated in RM. Use today's date.

<Field>

<Type>Text
<Name>SrcCitation
<Display>Ancestry Source Citation
<Detail>true    This is a Citation field.
<blank>

This is the full contents of the Ancestry provided Source Citation. Not all collections have this info. It is often not very helpful, other times it is crucial.

Enter the full text of the transcribed record in the Citation "Research Note" field
